复现 ctfshow 每周大挑战

附上官方wphttps://ctf-show.feishu.cn/docx/ToiJd70SboRn52xhn3WcJsfjnah

一

我们可以看到是过滤了(和.那么我们的system()是不能用了，直接是可以用反引号执行命令的姿势来使用code=echo ls /;

二

首先，我们可以看到过滤了很多东西 “/[a-zA-Z0-9@#%^&*:{}-"|`~\\]/“我们可以看看有什么没有过滤掉的

可知有! $ ‘ ( ) + , . / ; = [ ] _ 未被过滤，运用的是一种变量自增的方法来构造payload当变量赋值为[]时候，echo出来的是Array，当变量为[].时就是Array。然后echo出来就是A

那这个时候就有朋友要问了，为什么一定要[]._拼接呢？这是因为[]只是创造了一个空数组，由于数组为空，该元素不存在 → 返回 null。而[].则是Array拼接上_成为字符串才能用$[0]把A输出，0的话可以用条件判断的方式来替代。然后后续的步骤就是运用自增和拼接来创造了

三

细心的朋友发现了，这题加了一个if判断if (is_string($ctfshow) && strlen($ctfshow) 与上题还有什么区别呢，过滤的变了一点点

![](http://xendria.icu/wp-content/uploads/2025/07/c63f02a5-ee17-451f-80c2-d4bb778a215b-1024x263.png)

$ ( ) + , . / 0 1 ; = [ ] _ 很明显的多了几个没有被ban的虽然多了几个可用的字符但是对于长度的限制导致我们好像不能用上一个来整了(太长了)无处可寻翻了翻官方wp这边的思路是运用(0/0)和(1/0)来构造float中的NAN和INF来进行一个省略自增的数量，构造的话还是差不多

四

没被ban的有这几个$ ( ) + , . / 0 ; = [ ] _然后就是长度限制变成84了，然后就是尝试了

那我们该从哪里优化呢，主播想不出来了，然后看到官方wp中的(_/_._)[0];也是等于N的，真是优雅，利用(/._)为0的结果来直接构造出N，那么我们的payload也就能出来了

还有一种解法是说使用burp利用不可见字符%ff也能通过（我留着下一题写）

五

也是终于最后一题了，话不多说上代码

诶，长度限制变成了73，然后只剩下了$ ( ) + , . / ; = [ ] 这些可用，0居然也不能用了诶。那怎么办呢？这局应该就是这个phpinfo()该起作用的时候了官方wp说**这里观察到phpinfo安装了一个扩展gettext，该扩展支持函数_() ,相当于gettext()，直接转化为字符串。另外，其实数组下标使用未定义常量，php会warning，但是可以继续运行，并返回下标为0的字符（现象是这样但是实际机制需要看php源码）**就是说可用用()就等于gettext()我们可用_(/)[0]来构造N

然后就是把$a替换为不可见字符变量

对于这题官方wp还有更加简短的payload（师傅们真是太强辣！），可以配合这位师傅的文章一起食用https://blog.csdn.net/qq_38798840/article/details/130944199

小结

用了两天才磕磕碰碰做完了这五题，让我加深了自增这方面的印象吧（原来到处都可以加上这个pwp），也积累了这种题型，呜，自己还是有点太懒了，要勤奋一些了。
